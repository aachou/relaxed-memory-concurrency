# 共享内存一致性模型——总结

> **原标题：** Shared Memory Consistency Models: A Tutorial  
> **作者：** Sarita V. Adve, Kourosh Gharachorloo  
> **核心问题：** 多处理器共享内存系统中，读操作应返回哪个写操作的值？即内存一致性模型的定义、实现与权衡。

---

## 1. 顺序一致性（Sequential Consistency, SC）

Lamport 定义：任何执行的结果，都与所有处理器的操作按**某个顺序**逐个执行、且每个处理器的操作按**程序顺序**出现在该序列中的结果相同。

SC 有两个方面：

| 方面 | 含义 | 禁止的优化 |
|------|------|-----------|
| **程序顺序** | 单处理器内操作必须按程序顺序执行 | 写缓冲绕过、写操作重叠、非阻塞读 |
| **写原子性** | 写操作应对所有处理器瞬时可见 | 写通缓存提前返回、延迟传播失效/更新 |

SC 是**最直观**的模型，但也是**约束最强**的模型——它禁止了大量单处理器中常见的硬件和编译器优化。

---

## 2. 为什么 SC 太严格？

### 硬件优化受限制

| 优化 | SC 下的问题 | 对应放宽 |
|------|-------------|---------|
| 写缓冲绕过 | 读可能越过未完成的写，破坏 Dekker 算法 | 放宽写→读顺序 |
| 写操作重叠 | 写入不同地址可能乱序完成，破坏生产者-消费者模式 | 放宽写→写顺序 |
| 非阻塞读 | 读操作可能乱序完成，破坏同步逻辑 | 放宽读→读/读→写顺序 |

### 缓存引入额外复杂性

- 缓存一致性协议只是 SC 的**一部分**（保证同一位置的写操作串行化），不足以保证 SC
- 写操作完成需要等待所有副本确认失效/更新
- 写原子性假象需要额外机制（如两阶段更新）

### 编译器优化受限制

- 共享内存操作不能重排
- 寄存器分配不能将共享变量提升到寄存器（否则其他处理器的写入不可见）

## 3. 宽松内存模型全景

按放宽程度递进排列：

| 模型 | 放宽程序顺序 | 放宽写原子性 | 安全网 | 商用系统 |
|------|------------|------------|--------|---------|
| **IBM 370** | 写→读 | 否 | 串行化指令 | IBM 370 |
| **TSO** (SPARC V8) | 写→读 | 部分（自身写可见） | RMW 操作 | SPARC |
| **PC** (Processor Consistency) | 写→读 | 是（其他写也可见） | RMW 操作 | — |
| **PSO** (SPARC V8) | 写→读 + 写→写 | 部分（同 TSO） | STBAR + RMW | SPARC |
| **WO** (Weak Ordering) | 全部（数据操作之间） | 否 | 同步/数据分类 | — |
| **RCsc** | 全部（普通操作之间） | 复杂实现可破坏 | acquire/release 标记 | — |
| **RCpc** | 全部（普通操作之间） | 是 | acquire/release + RMW | — |
| **Alpha** | 全部 | 否 | MB / WMB 栅障 | DEC Alpha |
| **RMO** (SPARC V9) | 全部 | 否 | MEMBAR（4位编码） | SPARC V9 |
| **PowerPC** | 全部 | 是 | SYNC + RMW | PowerPC |

### 安全网（Safety Nets）三种模式

1. **显式栅障指令**：MB/WMB (Alpha)、MEMBAR (RMO)、SYNC (PowerPC)、STBAR (PSO)
2. **操作分类标记**：WO 的同步/数据、RC 的 acquire/release
3. **RMW 操作**：在缺乏显式 fence 或标记机制时，利用 RMW 的原子性间接恢复顺序

## 4. 以程序员为中心的视角（DRF0 / PL）

核心思想：与其让程序员理解底层重排细节，不如让程序员提供**程序级信息**，由系统保证优化不破坏正确性。

### 无数据竞争（DRF0）模型

- 程序员将所有**可能参与竞争**的操作标记为同步操作
- 剩余操作标记为数据操作（可自由重排）
- 在一个顺序一致执行中，如果两个操作访问同一位置、至少一个是写、且之间无其他操作插入，则它们构成**竞争**
- 正确标记后，系统保证程序行为等价于顺序一致性
- 现代 C++ 内存模型（atomic/fence/mutex）本质上是 DRF0 的继承者

### 信息传递机制

| 层次 | 机制 |
|------|------|
| 语言级 | doall 循环隐式无竞争、同步库调用、显式类型声明（如 C++ `memory_order`） |
| 硬件级 | 专用操作码、虚拟地址高位编码（address shadowing）、栅障指令翻译 |

---

## 5. 关键结论

| 维度 | 要点 |
|------|------|
| **SC 的代价** | 禁止大量优化，直接实现性能差；但可通过预取和推测执行部分缓解 |
| **宽松模型的收益** | 启用硬件写延迟隐藏、读延迟隐藏、编译器优化，性能显著提升 |
| **编程复杂度** | 以系统为中心的规范暴露底层优化细节，增加理解难度 |
| **DRF0 的价值** | 将复杂度转移到系统侧，程序员只需正确标记同步操作 |
| **对现代语言的影响** | C++11 atomics、Rust Ordering、Java volatile 均根源于此框架 |
