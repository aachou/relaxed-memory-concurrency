<h2 align="center">A Correctness Condition for Concurrent Objects</h2>
<p align="center">Maurice P. Herlihy & Jeannette M. Wing, CMU</p>

## 1. 为什么需要线性一致

两个线程对同一个共享计数器各执行一次 `x += 1`。期望结果是 2，但如果两个线程同时读到了 0：

```
线程 A: 读 x → 0
线程 B: 读 x → 0
线程 A: 写 x ← 1
线程 B: 写 x ← 1
```

结果是 1。任何顺序执行（A 做完 B 再做，或反之）都只会得到 2。

问题不是代码写错了——单线程下它从不出错。问题是：没有一个标准来**判断一个并发执行是否可以被认为是正确**的。Herlihy & Wing 给出的答案就是线性一致性。

## 2. 线性一致的定义

### 2.1  顺序规约

每个数据类型定义了哪些操作序列是合法的，这叫**顺序规约**。例如：

- 栈：Pop 返回最近 Push 且尚未 Pop 的值（LIFO）
- 队列：Deq 返回最早 Enq 且尚未 Deq 的值（FIFO）
- 寄存器：Read 返回最近一次 Write 写入的值

### 2.2 历史

一次并发运行的全部事件按时间排成序列，称为**历史**。每个事件是某个操作的 invocation（调用）或 response（返回）。

```
时间 →
线程 A:   Enq(1)调用 ────── Enq(1)返回
线程 B:              Deq()调用 ────────── Deq()返回1
```

- 从调用到返回的区间是操作"正在执行"的时间段
- 区间重叠的两个操作叫并发操作，不重叠的叫非并发操作

### 2.3 可线性化

一个历史要被称为**可线性化**的，需要存在一个合法的顺序执行 S，使得 S 的执行结果与原历史一致，且保留原历史中非并发操作的先后顺序。

S 称为这个历史的线性化。

### 2.4 线性化点

线性化点是一个操作调用之后、返回之前执行成功的点。线性点代表了操作本身，所以找线性化点并按时间排序就等价于构造这些操作的一个顺序执行 S。剩下的唯一问题是：按这些线性化点排序得到的 S 是否符合该数据类型的顺序规约？

- 符合 → 这个历史是可线性化的
- 找不到这样的线性化点，或任何排序结果都不符合顺序规约 → 这个历史是不可线性化的

### 2.5 数据结构满足线性一致

一个数据结构满足线性一致性，当且仅当它所有可能的并发历史都是可线性化的。

## 3. 证明 Treiber Stack 是线性一致的

```
push(x):
  n = new Node(x)
  loop
    t = top
    n.next = t
    if CAS(&top, t, n): return

pop():
  loop
    t = top
    if t == null: return EMPTY
    n = t.next
    if CAS(&top, t, n): return t.val
```

**线性化点**：push 和 pop 的线性化点都是成功的 CAS 指令那一刻。失败的 CAS 没有效果，没有线性化点。

按成功 CAS 的时间排序得到操作序列。每个 push 的成功 CAS 在栈顶加一个新节点；每个 pop 的成功 CAS 移除栈顶节点并返回它的值。这个序列符合 LIFO——最后 push 的最先被 pop。这说明按线性点排序得到的 S 满足栈的顺序规约，因此 Treiber Stack 的所有并发历史都是可线性化的——它满足线性一致性。

> 注：实际实现还需解决 ABA 问题，配合 hazard pointer 或 EBR 管理内存回收。这是工程问题，不影响线性一致性判断。

## 4. 与其他一致性模型对比

| | 线性一致性 | Sequential Consistency | Serializability |
|---|---|---|---|
| 保留非并发操作的实时顺序 | 是 | 否 | 否 |
| 可组合（local） | 是 | 否 | 否 |

**实时顺序**：如果 A 返回后才调用 B，线性一致性要求 S 中 A 必须在 B 之前。SC 和 Serializability 都没有这个约束——即使 A 执行完很久了，B 仍可能看到 A 没发生之前的值。

**可组合性**：线性一致性是局部的——只要每个对象满足线性一致，那么由这些对象构成的系统也是线性一致的。两个 SC 对象放一起，整体不一定 SC。

## 5. 总结

线性一致性（Linearizability）是 Herlihy & Wing 提出的并发对象正确性条件。它要求**每个操作在其调用之后、返回之前的某个瞬间点原子地完成（这个点称为线性化点），所有非并发操作的先后顺序必须保留**。判断一个并发执行是否正确，就是给每个操作找线性化点并排序，看结果是否符合数据类型本身的顺序规约。

这篇论文是并发理论的基石，奠定了并发对象正确性的标准。
